HTML5 Elements Learned in Lab5

Instructions

Implement each of the new elements you learned in lecture in your newly created document. Create a sound document that has the common elements (h1, h2, p, etc.), is semantic and well-structured, and contains comments.

  1. <abbr> Explains the definition of an abbreviation.

    ie: This is what WHATWG abbreviation means.

  2. <address> Represents the contact information for its enclosing section. If it is a child of the body element, then it applies to the document as a whole. And would appear usually in a footer for like contact email or links. Not to used for physical address.

    ie:

    Dave Raggett, Arnaud Le Hors,

    are contact persons for the W3C HTML Activity

  3. <b> Represents a span of text to be stylistically offset from the normal prose without conveying any extra importance. Examples are key words in a document abstract, product names in a review, or other spans of text whose typical typographic presentation is bold.

    ie: Hell yea!

  4. <i> Represents a span of text in an alternate voice or mood, or otherwise offset from the normal prose, such as a taxonomic designation, a technical term, an idiomatic phrase from another language, a thought, a ship name, or some other prose whose typical typographic presentation is italicized.

    ie: Last night I ate Pizza!.

  5. <code> The code element represents a fragment of computer code. This could be an XML element name, a filename, a computer program, or any other string that a computer would recognize. It is being used at the beginning of every li.

    ie: Every html document needs <doctype> at the top of the page.

  6. <del> The del element represents a removal from the document. The del elements should not cross implied paragraph boundaries

    ie: My favorite colour is Red yellow. Riiight?

  7. <ins> The ins element represents an addition to the document. The ins elements should not cross implied paragraph boundaries.

    ie: My favourite colour is blue red, but I also like green and yellow.

  8. <details> The details element represents a disclosure widget from which the user can obtain additional information or controls. However, the details element is not appropriate for footnotes. Currently is only supported in Chrome.

    ie:

    Name of a good magazine

    Wired Magazine

  9. <summary> The summary element represents a summary, caption, or legend for the rest of the contents of the summary element's parent details element, if any.

    ie:

    More information

    Here is the source data that is discussed in the article ...

  10. <dfn> The dfn element represents the defining instance of a term. The paragraph, description list group, or section that is the nearest ancestor of the dfn element must also contain the definition(s) for the term given by the dfn element.

    ie: While they're essential reading material for our job, the W3C specifications are not exactly George R. R. Martin-level reading material.

    ie:

  12. <figcaption> The figcaption element represents a caption or legend for the rest of the contents of the figcaption element's parent figure element, if any.

    ie:

  14. <kbd> The kbd element represents user input (typically keyboard input, although it may also be used to represent other input, such as voice commands).

    ie: Mac users: To take a screenshot press of a certain program Command+Shift+4+Space

  15. <mark> Represents a run of text in one document marked or highlighted because of its relevance in another context.

    ie: Essentially mark is a highlighter. Can you see where it has been used?

  16. <meter> When the meter binding applies to a meter element, the element is expected to render as an 'inline-block' box with a 'height' of '1em' and a 'width' of '5em', a 'vertical-align' of '-0.2em', and with its contents depicting a gauge.

    ie: Storage space usage: 6 blocks used (out of 8 total) Voter turnout: Tickets sold:

  17. <pre> The pre element represents a block of preformatted text, in which structure is represented by typographic conventions rather than by elements.

    ie: The pre preserves the poems formatting, which forms an intrinsic part of the poem itself

  18. <samp> The samp element represents (sample) output from a program or computing system

    ie: 

    mike:mysite mike$ git status

						nothing to commit (working directory clean)

  19. <small> Represents side comments such as small print. It is not intended to be presentational. The small element should not be used for extended spans of text such as multiple paragraphs, lists, or sections of text. It is only intended for short runs of text.

    ie: where the copyright is small

  20. <sub> The sub element can be used inside a var element, for variables that have subscripts.

    ie: H2O is the chemical formula for water

  21. <sup> The sup element represents a superscript and the sub element represents a subscript.

    ie: Today is the 5th of December

  22. <time> Represents a precise date and/or time in the proleptic Gregorian calendar. The time element encodes modern dates and times in a machine-readable way, so that, for example, user agents could offer to add an event to the user's calendar.

    ie: 2011-11-12T14:54

  23. <var> The var element represents a variable. This could be an actual variable in a mathematical expression or programming context, or it could just be a term used as a placeholder in prose.

    ie: E energy is equal to m mass multiplied by the c speed of light, squared.
